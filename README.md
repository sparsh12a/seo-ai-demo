# AISEO — AI-Powered SEO Article Generator

Generate fully researched, SEO-optimized articles from a single keyword. The pipeline runs SERP analysis, keyword extraction, outline generation, article writing, and quality scoring — all automatically.

---

## What it does

1. **SERP Analysis** — fetches top Google results for your keyword via SerpAPI
2. **Keyword Extraction** — identifies primary and secondary keywords using Claude AI
3. **Outline + Article Generation** — builds a structured outline then writes the full article in Markdown
4. **Quality Check** — scores the article and triggers a revision if it falls below threshold
5. **SEO Metadata** — generates a title tag, meta description, internal linking suggestions, and external references
6. **REST API** — FastAPI layer exposes the pipeline over HTTP so the frontend can submit jobs and retrieve results
7. **Next.js Frontend** — clean UI to generate articles, view results, and browse history

## Setup

### 1. Clone & create environment

```bash
git clone <repo-url>
cd seo-ai
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 2. Configure API keys

```bash
cp .env.example .env
```

Edit `.env` and fill in your keys:

```
ANTHROPIC_API_KEY=your_anthropic_api_key_here
SERPAPI_KEY=your_serpapi_key_here
DB_PATH=seo_jobs.db
```

### 3. Start the backend

```bash
uvicorn api:app --reload
# Runs on http://localhost:8000
```

### 4. Start the frontend

```bash
cd frontend
npm install
npm run dev
# Runs on http://localhost:3000
```

### 5. Add your API keys (first time)

On first login, you'll be redirected to the **API Keys** page (`/settings`). Enter your:
- **Anthropic API key** — from [console.anthropic.com](https://console.anthropic.com)
- **SerpAPI key** — from [serpapi.com](https://serpapi.com)

Keys are stored in secure httpOnly cookies and expire after 1 hour. You'll be asked to re-enter them after each session. They are never stored on the server or in the database.

## Design decisions

**Single-pass pipeline with revision loop** — The orchestrator runs each stage sequentially (SERP → keywords → outline → article → quality check). If the quality score falls below threshold, only the article generation step is retried (up to 3 times), not the expensive SERP fetch.

**Structured JSON outputs from Claude** — Every Claude call returns strict JSON validated against a Pydantic model. This makes each stage independently testable and prevents hallucinated structure from propagating downstream.

**SQLite for job storage** — Keeps the setup dependency-free (no Postgres, no Redis). Each job's full output is stored as a JSON blob so any past article can be retrieved instantly without re-running the pipeline.

**Synchronous HTTP** — `POST /jobs` blocks until the article is ready (~30–60s). This keeps the architecture simple — no job queues, no polling complexity — which is appropriate for a single-user tool.

**FAQs from SERP data** — The FAQ section is generated by analyzing the "People Also Ask" questions present in SERP snippets, grounding the content in what real users are actually searching for.

## Tests

```bash
pytest tests/
```

Covers SEO constraint validators:
- `has_h1` — article must have an H1
- `keyword_in_title` — primary keyword present in H1
- `keyword_in_intro` — keyword appears in first section content
- `word_count_in_range` — within ±20% of target word count
- `min_heading_count` — at least 5 headings for structure

## API

| Method | Path | Description |
|---|---|---|
| `GET` | `/health` | Liveness check |
| `POST` | `/jobs` | Generate article (synchronous, ~30–60s) |
| `GET` | `/jobs` | List all past jobs |
| `GET` | `/jobs/{job_id}` | Get a specific job's output |

**Example:**
```bash
curl -X POST http://localhost:8000/jobs \
  -H "Content-Type: application/json" \
  -d '{"keyword": "best productivity tools", "word_count": 1500}'
```
